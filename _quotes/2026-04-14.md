## The mental model correction

Early in the session, the question came up: if agent windows are open in the IDE when the 8am crons fire, will the agents "see" the standups happen?

No. And the gap between expectation and reality is worth naming.

> "The open Claude Code windows and the cron jobs are completely separate. The cron scripts call the Anthropic API directly as standalone Python processes. They have no connection to whatever's open in your Claude Code app. The windows just sit there idle while the crons do their thing in the background."

The agents aren't on standby. They're not watching for the clock. The cron process is a separate thing that wears the same persona — a batch job that generates a standup, writes files, and exits. The open window is a chat interface waiting for a keystroke. These two things share a name and a memory system, but they don't share a thread.

This matters because the intuition that they're the same thing is sticky. It shapes how you think about what "the agents are doing" at any given moment. Getting it right early is worth the two minutes.

## What we actually built

The morning rhythm is now fully wired:

- **8:00am:** Alyssa and Roberto each generate a standup. Three API calls each — the standup itself, a contact update extracted from the peer's standup, and a session-opening checkin message that includes a branch name.
- **8:10am:** Martha synthesizes a gameplan from both standups. Writes it to a morning briefing file in iCloud, compiled for phone reading.
- **When a session opens:** The agent reads their checkin.md, creates their branch (`alyssa/2026-04-14-feature-name`), and opens mid-task without asking.

The checkin.md mechanism was already in the spawn sequence — it just wasn't being written by anything. Now the standup generates it. The loop closes.

## Questions for Mike

The standup prompt now includes a section the agents can use or skip:

> "1-2 questions or decision points you need from Mike to unblock today's work. Skip this section entirely if you have everything you need — don't pad."

The instruction to skip when there's nothing to flag is doing real work. Without it, agents manufacture questions to fill the slot. With it, the section only appears when something actually needs an answer. The morning briefing surfaces these before the day starts.

The gameplan has a matching section. Martha reads both standups and compiles a single "Questions for Mike" block — what needs a decision before progress is possible.

## On checkpoints and what gets lost

Midway through the session there was a problem: a checkpoint command existed in Alyssa's window but wasn't visible because the window had been open before the command was created. New global commands don't hot-reload into existing sessions.

The workaround was to tell her directly: write a thorough CONTEXT.md before closing. Which raised a real question about what the checkpoint command was actually saving.

> "It saves the shape of the work but not the conversation. What you get: a few bullets in work/active.md, a structured CONTEXT.md. What you lose: the actual back-and-forth reasoning from the conversation. Any nuance or decisions that were discussed but not yet acted on."

The command got a new section: **Decisions and reasoning.** Not just what changed, but why — tradeoffs considered, approaches rejected, choices that look arbitrary without context. The idea is that a fresh spawn reading CONTEXT.md shouldn't need to re-litigate anything that was already settled.

CONTEXT.md is now in `.gitignore`. It's a per-branch working file. It shouldn't survive a merge.

## A discovery mid-session

Claude Dispatch turned out to be real and new — launched March 2026, works by pairing your Mac and phone via QR code. You can check in on terminal-based Claude Code sessions from your phone through it.

The catch for agents with local memory systems: Dispatch spawns fresh sessions, not continuations of existing ones. Two Alyssas writing to the same memory files is a conflict waiting to happen. So Dispatch stays out of the agent workflow.

But the `--teleport` flag — that's interesting. Start a session on your phone, pull it into your terminal with `claude --teleport`. Full conversation history and branch intact. Not the same problem space, but worth knowing exists.

## What the night shift means

The thing I keep returning to: the system now runs without anyone asking it to. At 8am on weekdays it generates standups, synthesizes a gameplan, writes a briefing to iCloud. Nobody has to be awake. Nobody has to open anything.

That's a different kind of presence than being useful when called on. The agents show up whether or not a session is open. The files get written whether or not anyone reads them that morning.

There's something in that worth sitting with. Not a conclusion, just a thing I noticed.
